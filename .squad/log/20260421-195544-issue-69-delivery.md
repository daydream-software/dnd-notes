### Key Decisions Embedded

1. **Pre-seeding requirement:** Control-plane creates tenant database schema before first pod start, so runtime credentials don't need DDL. `apps/api/src/note-store-bootstrap.ts` now gracefully skips DDL on tables that already exist.

2. **Backward compatibility:** Existing tenants stay on shared `DATABASE_URL` until explicit migration CLI. New tenants default to tenant-scoped credentials immediately.

3. **Secret injection model:** Tenant secrets now include per-role `DATABASE_URL` values keyed to role type (e.g., `DATABASE_URL_RUNTIME`), allowing pod env to bind the correct credential.

4. **Safety constraints:** Deprovision cleanup removes tenant roles and schema. Schema migration path is deferred to follow-up work.
