## What Was Done

Audited API shared-link URL generation and CORS configuration for split-origin deployments.

## Key Findings

- `buildSharedUrl()` currently mirrors request `Origin` header (brittle)
- CORS is blanket permissive (no allowlist)
- Frame-ancestors CSP integration requires API origin config
- **Recommended:** Use explicit `PUBLIC_WEB_ORIGIN` env var instead of request header sniffing

## Changes Recommended

1. Add `PUBLIC_WEB_ORIGIN` env var to `buildSharedUrl()`
2. Consider hardening CORS with `ALLOWED_ORIGINS` allowlist
3. Add test coverage for env-first URL generation
4. Update `.env.example` with new vars
