### Deliverables

1. **Acceptance Criteria (AC1–AC7)**
   - AC1: Search input & campaign scope
   - AC2: Title/body matching (case-insensitive, partial word, special char literal)
   - AC3: Tag filtering (AND logic, tag count accuracy, scope isolation)
   - AC4: Session filtering (null session handling, campaign-scoped)
   - AC5: Collaborator filtering (creator vs. editor distinction)
   - AC6: Mobile responsive behavior
   - AC7: Integration with existing flows (quick capture, session browsing, activity view)

2. **Regression Test Matrix (30+ cases)**
   - Autocomplete deduplication under concurrent edits
   - Empty campaign handling (no notes, no tags, no collaborators)
   - Special character handling (%, &, quotes, etc.)
   - Tag facet count updates under concurrent note creation/deletion
   - Multi-tag AND filtering correctness
   - State persistence across mode switches (browse ↔ search ↔ sessions ↔ activity)
   - Stale-response protection (rapid clicks on same tag)
   - Create-note flow while filtered (does new note appear immediately?)
   - Mode-switching orthogonality (toggling search/session/activity modes)
   - Issue #27 regression: no accidental state-reload during mode switch
   - Null session notes + valid session notes in same search result
   - Collaborator filter with claimed/unclaimed members
   - Tag facet behavior when a tag is deleted or all notes lose a tag

3. **UX Blockers & Product Sign-Off Checklist (7 items)**
   - Tag name normalization (case-sensitive or insensitive display?)
   - Multiple-tag logic (AND vs. OR; recommendation: AND = more precise)
   - Autocomplete trigger timing (on first char? on backspace?)
   - Facet sort order (by count desc, or alphabetic, or by recency?)
   - Empty-state copy & design (empty search, no matches, no campaign tags, no collaborators)
   - Null session handling (show "Unassigned" or "No Session"?)
   - Count staleness tolerance (how fresh must tag counts be during rapid edits?)
