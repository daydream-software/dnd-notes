## What Was Done

Synthesized findings from Stef (frontend), Data (backend), and Brand (platform) audits into a unified decision on shared-link URL generation and origin-model.

## Decision

Treat production shared-link generation as a backend-owned canonical URL problem:
- Add explicit `PUBLIC_WEB_ORIGIN` (or `PUBLIC_WEB_URL`) env var
- Prefer same-origin deployment as default target shape
- Do not couple link generation to request `Origin` header

## Implementation (Thin Slice)

1. Add API env parsing for `PUBLIC_WEB_ORIGIN` in startup
2. Change `buildSharedUrl()` to prefer env var over request header
3. Keep fallback for local/dev compatibility
4. Add tests for env-first generation + fallback behavior
5. Document in README and `.env.example`

## Cross-Team Impact

- Simplifies deployment (no CORS allowlist needed unless split-origin)
- Stabilizes shared-link URLs (not vulnerable to proxy/forwarding issues)
- Clear boundaries: same-origin is default, split-origin is opt-in
