## Assignment: Issue #33 Frontend (Recent Activity UI)

**What:** Implement read-only activity feed UI for recent notes, sorted by modification time, with per-collaborator filtering.

**Scope:** Thin slice v1
- Activity list view, sorted by `updatedAt` descending
- Collaborator sidebar with filter (click to filter, click again to clear)
- Distinguish 'created' vs 'edited' actions with timestamps and actor attribution
- Empty state message
- Membership-aware access (not auth-only, supports linked collaborators)

**Not in scope (future):**
- Pagination / "Load more" UI
- Full-text search / tag filtering
- Shared workspace activity support (product decision pending)
- Session filtering / activity diffs

**Backend contract (stable):**
```
GET /api/notes/activity?campaignId=...&membershipId=...&limit=20
Response:
{
  campaign: CampaignSummary,
  collaborators: [{ membershipId, displayName, role, noteCount }],
  activity: [{
    id, campaignId, title, body, tags, status, sessionName,
    action: 'created' | 'edited',
    createdBy: { membershipId, displayName, role },
    lastEditedBy: { membershipId, displayName, role } | null,
    createdAt, updatedAt
  }]
}
```

**Files to modify:**
- `apps/web/src/App.tsx` — Activity tab UI, collaborator filter state, mode toggle
- `apps/web/src/api.ts` — `fetchActivity()` client function
- `apps/web/src/types.ts` — Activity response types
- `apps/web/src/App.test.tsx` — Regression tests (draft preservation, stale-response handling, empty state)

**Regression gates (critical):**
1. Mode toggle (activity ↔ all notes ↔ session browse) does NOT trigger `loadWorkspace()` re-fetch
2. Collaborator filter does NOT trigger full workspace reload
3. Unsaved note drafts survive mode switches
4. Rapid collaborator filter clicks show latest selection only (no heading/list mismatch)
5. Empty state renders cleanly when campaign has no notes

**Test coverage:**
- Web regression suite: 3+ new tests (mode toggle, draft survival, stale-response handling)
- Integration: Activity + membership auth (claimed collaborators can access linked campaigns)

**Expected output:**
- PR with activity UI merged to main
- All regression tests passing
- Chunk (tester) approval required

**Timeline:** 2–3 day slice post-PR-#36 merge (depends on rebase completion)
