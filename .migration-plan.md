# Control Plane Postgres Migration Plan

## Goal
Migrate control-plane registry from SQLite to Postgres

## Current State
- Control-plane uses better-sqlite3 for registry (tenant metadata, portal accounts, sessions)
- DATABASE_PATH env var points to /app/data/control-plane.sqlite
- PVC dnd-notes-control-plane-data mounted at /app/data
- Smoke tests run control-plane locally with SQLite

## Target State
- Control-plane uses pg (Postgres) for registry
- CONTROL_PLANE_DATABASE_URL env var for Postgres connection
- PVC removed from manifests
- Smoke tests updated to use in-cluster Postgres

## Implementation Steps
1. Add CONTROL_PLANE_DATABASE_URL support to tenant-registry
2. Keep better-sqlite3 as fallback for local dev (DATABASE_PATH)
3. Update manifests to use Postgres secret
4. Remove PVC from base manifests
5. Update smoke scripts to wire Postgres
6. Update tests to support both backends
7. Run validation

## Acceptance
- `npm test --workspace apps/control-plane` passes
- `npm run k3d:full-stack-smoke` passes with control-plane on Postgres
